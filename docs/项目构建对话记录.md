## 6. Supabase 接入过程中的问题

### 6.1 邮箱注册限流

**你：**  
为什么我注册邮箱的时候显示 `email rate exceed`？

**我：**  
解释这是 Supabase 默认邮件服务的限流，不是代码错误。

给出的处理方式：

- 等待冷却时间
- 不要频繁重复点击注册/魔法链接
- 正式环境建议配置自定义 SMTP
- 联调阶段也可临时关闭邮箱确认

---

### 6.2 邀请码接受时报错

**你：**  
接受对方邀请时出现：`column reference "space_id" is ambiguous`

**我：**  
定位到是 Supabase SQL 函数 `accept_invitation()` 中存在字段名歧义。

修复方式：

- 重写函数中的查询
- 改为使用表别名，例如 `sm.space_id`

并提供了可直接复制到 Supabase SQL Editor 执行的补丁。

---

### 6.3 创建回忆时报错

**你：**  
创建回忆时出现：`Could not find a relationship between 'memories' and 'profiles' in the schema cache`

**我：**  
分析后确认：

- 前端查询是按 `profiles!memories_created_by_fkey` 这种关系名写的
- 但数据库的外键关系名与目标表不一致

后续我帮助修正了：

- `space_members`
- `memories`
- `anniversaries`
- `notes`
- `wishlists`
- `activity_logs`

这些表的 profile 关系和外键命名，使其与前端查询完全对齐。

## 9. 功能迭代：纪念日添加问题修复

### 第一轮修复

**我：**  
先修正了纪念日表单的用户反馈问题：

- 默认自动填入当天日期
- 提交失败时显示校验错误
- 提交成功后刷新纪念日和动态列表

文件：

- [components/forms/anniversary-form.tsx](/Users/kuangjiechen/Desktop/小猪APP/components/forms/anniversary-form.tsx)

### 第二轮排查

**你：**  
评论功能已经可以使用了，但添加纪念日时仍会出现：明明已经选择了纪念日类型，点击添加时还是提示“请先补全纪念日信息”。

**我：**  
继续定位后发现两个关键点：

1. `Select` 组件没有用 `forwardRef`
2. 纪念日除了类型，还要求填写标题；用户虽然选了类型，但标题字段仍可能是空的

### 第二轮修复内容

- 把 `Select` 改成标准的 `forwardRef` 写法，保证 `react-hook-form` 稳定读取值
- 选择预设类型后，自动带出默认标题：
  - 恋爱开始日
  - 生日
  - 第一次见面
  - 第一次旅行
- 提交失败时优先弹出具体字段错误，而不是笼统提示

主要文件：

- [components/ui/select.tsx](/Users/kuangjiechen/Desktop/小猪APP/components/ui/select.tsx)
- [components/forms/anniversary-form.tsx](/Users/kuangjiechen/Desktop/小猪APP/components/forms/anniversary-form.tsx)
